# 通用模板

本文件定义了各角色产出文档的通用结构，确保文档风格统一、内容规范。

---

## 计划/任务列表通用结构

适用于：迭代计划、进度总览、任务分配等文档。

```markdown
# {文档标题} - {版本/日期}

## 目标
{一句话描述本计划的目标}

## 当前状态
{状态描述，如：规划中 / 进行中 / 已完成 / 已归档}

## 任务列表

### 进行中
- [ ] {任务标题}
  - 负责人: {角色}
  - 截止: {时间或迭代}
  - 状态: {进行中/阻塞}
  - 说明: {简短说明}

### 待开始
- [ ] {任务标题}
  - 负责人: {角色}
  - 依赖: {前置任务}

### 已完成
- [x] {任务标题}
  - 完成时间: {时间}
  - 产出: {文档路径或说明}

## 阻塞事项
| 事项 | 阻塞原因 | 待决策方 | 状态 |
|------|----------|----------|------|
| {事项} | {原因} | {角色/人类} | 待处理 |

## 下一步
1. {下一步行动}
2. {下一步行动}
```

## 版本化报告通用结构

适用于：测试报告、质量报告、代码审查报告等。

```markdown
# {报告类型} - {版本/日期}

## 概述
- **测试/审查对象**: {功能/模块名称}
- **执行时间**: {时间}
- **执行人**: {角色}
- **结论**: ✅ 通过 / ❌ 不通过 / ⚠️ 有条件通过

## 详细结果

### 通过项
- [x] {检查项描述}
- [x] {检查项描述}

### 失败项
- [ ] {检查项描述}
  - 问题: {具体描述}
  - 严重级别: P0 / P1 / P2 / P3
  - 建议: {修复建议}

### 跳过项（如有）
- [ ] {检查项描述} — 跳过原因: {原因}

## 统计摘要
| 指标 | 数值 |
|------|------|
| 总检查项 | {n} |
| 通过 | {n} |
| 失败 | {n} |
| 跳过 | {n} |
| 通过率 | {xx%} |

## 结论与建议
{综合结论，以及下一步建议}
```

## 复盘报告通用结构

适用于：迭代复盘、修复循环复盘等。

```markdown
# 复盘报告 - {迭代版本/任务名称}

## 基本信息
- **任务/迭代**: {名称}
- **时间范围**: {开始} ~ {结束}
- **最终状态**: 成功 / 失败 / 部分完成

## 完成情况
- **目标达成度**: {百分比或说明}
- **完成事项**: {列表}
- **未完成事项**: {列表及原因}

## 效率分析
- **各阶段耗时**: {分布说明}
- **阻塞事件**: {次数和主要原因}
- **返工次数**: {次数和原因}

## 质量分析
- **发现问题数**: {总数，按 P0/P1/P2/P3 分类}
- **修复率**: {已修复 / 总数}
- **一次通过率**: {首次审查通过的比例}

## 协作评估
- **顺畅的环节**: {举例说明}
- **卡点/阻塞**: {具体描述}
- **职责边界**: {是否清晰}

## 改进建议

### 对现有流程的优化
1. **{流程/技能名称}**
   - 问题: {观察到的不足}
   - 建议: {具体改进方向}
   - 预期收益: {改善效果}

### 新增能力建议
- **建议新增**: {能力或技能名称}
- **触发场景**: {何时使用}

### 流程优化建议
- {具体流程调整建议}

## 经验教训
{可复用的经验要点，写入经验库}
```

## 问题跟踪条目结构

适用于：`.vibe/docs/问题跟踪.md` 中的各类条目。

### 求助条目
```markdown
## 求助-{编号}: {标题}
- **状态**: 待分析
- **提交人**: {角色}
- **描述**: {问题详情}
- **已尝试**: {至少3种已尝试的方法}
- **代码位置**: {文件路径:行号}
- **错误信息**: {如有}
```

### 分析结果条目
```markdown
## 分析-{编号}: {标题}
- **状态**: 待修复
- **根因**: {根本原因分析}
- **涉及文件**: {文件路径}
- **修复建议**: {具体修复方向}
- **注意事项**: {修复时的注意点}
```

### Bug 修复条目
```markdown
## BUG-{编号}: {标题}
- **状态**: 待验证 / 已验证
- **严重级别**: P0 / P1 / P2 / P3
- **描述**: {问题描述}
- **修复说明**: {修复内容}
- **验证方法**: {如何验证修复有效}
```
